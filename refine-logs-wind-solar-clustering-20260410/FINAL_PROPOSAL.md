# Final Proposal

**Topic**: 面向调度代价与极端风险保持的风光多视角风险约束协同聚类
**Date**: 2026-04-10

## Problem Anchor

高比例风光系统中的聚类方法，通常服务于典型场景提取、预测预处理或站群分区，但现有目标函数往往只保留统计相似性，不能确保关键的运行风险被保留下来。我们要解决的核心问题不是“如何把曲线分得更像”，而是：

**如何在强压缩条件下，仍然保留风光出力对系统最关键的调度与风险结构？**

## Method Thesis

将风光样本表示为多视角特征，并将“调度代价/风险保持”直接并入聚类目标函数，构造一个兼顾统计相似性、跨区域相关性和系统任务保持性的协同聚类方法。

## Dominant Contribution

1. 提出一个 **dispatch-aware / risk-preserving clustering** 目标，而不是普通相似性聚类。
2. 设计统一的多视角特征表示：输出形态、波动/爬坡、源荷失配、跨区相关。
3. 建立“聚类质量”与“下游系统误差”之间的评测协议，证明传统聚类在极端小时上失真更严重。

## Proposed Method

### Input object

- 对象可以是 `样本日`、`样本周` 或 `站点-日` 片段。
- 每个对象包含：
  - 风电出力序列
  - 光伏出力序列
  - 可选负荷序列
  - 地区或站点标签

### Multi-view representation

- `View 1`: 归一化风光联合出力形状
- `View 2`: 爬坡、波动、持续低出力时段等风险特征
- `View 3`: 残余负荷、源荷失配、净负荷尾部
- `View 4`: 跨区域相关、互补性、同步低出力事件

### Clustering objective

总目标由三部分构成：

- `Intra-cluster fidelity`: 保证簇内样本在多视角上相似
- `Risk preservation`: 保证聚类后重构样本对尾部风险与爬坡分布失真小
- `Task consistency`: 保证下游简化调度器或代理指标上的误差小

## Minimal Claim

在相同聚类数下，所提方法比常见风光聚类基线更能保留极端低出力、残余负荷尾部与调度代理成本，因此更适合作为高比例风光系统的代表场景生成器。

## Key Risks

- 如果没有负荷与下游任务，任务导向主张会变弱。
- 如果多视角太多，可能被认为是特征堆叠。
- 如果改进仅出现在少数指标上，贡献表达要聚焦“风险小时保持”。

## Scope Control

- 第一版不做复杂市场出清或完整 UC。
- 第一版不做端到端 GNN。
- 第一版先用可解释、可复现的聚类主干和轻量代理调度器。

## First Implementation Target

先完成“样本日级别”的风光联合聚类，并在 3-4 个典型指标上验证保持效果；若结果成立，再扩展到区域/站群动态聚类版本。
